# 鸿蒙应用需求分析 Skill

## 目的

将用户的业务需求转换为具体的技术实现点，避免无效检索。这是鸿蒙应用开发流程的第一步。

## 核心原则

**不要直接搜索业务词汇，要分析背后需要的技术组件。**

## 📋 需求分析模板

对于任何开发需求，先进行以下分析：

```python
REQUIREMENT_ANALYSIS = {
    "用户需求": "用户的原始描述",
    "界面分析": "需要什么UI组件？(列表、按钮、输入框、图片等)",
    "数据分析": "需要什么数据结构？(数组、对象、状态管理等)",
    "交互分析": "需要什么用户交互？(点击、滑动、输入等)",
    "问题类型": "UI开发/数据处理/语法查询/构建错误"
}
```

## 🔧 技术关键词细化策略

### 识别核心API组件

- 提取需要使用的具体技术组件名称
- 使用英文API名称作为查询关键词
- 一个关键词可以匹配到该组件所有相关内容

### 细化示例

```python
# 原始需求: "创建登录页面"
# 界面分析: 输入框、登录按钮、垂直布局
# 细化后关键词:
["Button", "TextInput", "Column"]
```

### 常见组件映射

| 业务需求 | 技术组件 |
|---------|---------|
| 列表展示 | List, ForEach |
| 按钮点击 | Button, onClick |
| 文本输入 | TextInput, onChange |
| 图片显示 | Image |
| 垂直布局 | Column |
| 水平布局 | Row |
| 滚动视图 | Scroll |
| 状态管理 | @State, @Observed |

## ✅ 正确的 L0 分析要点

1. **不做业务联想** - 不搜索"篮球"、"电商"、"游戏"等业务词汇
2. **专注技术组件** - 识别 UI 组件、数据结构、交互方式
3. **清晰分类** - 确定问题类型（UI开发/数据处理/语法查询/构建错误）

## ❌ 错误示例

### ❌ 错误分析 1
```
用户需求: "电商购物页面"
错误分析: 需要 PayPal 集成、库存管理系统、物流跟踪
问题: 直接联想业务场景，未分析技术实现点
```

### ❌ 错误分析 2
```
用户需求: "篮球排行榜"
技术关键词: ["排行榜", "排名", "得分"]
问题: 使用业务词汇，"排行榜"不是技术组件
正确做法: ["List", "Text", "数据排序"]
```

## 🛑 L0 评估

- ✅ 已分析出具体技术组件 -> **先使用 skill-B 中的仓颉语言 skills 进行开发**
- ❌ 需求太模糊无法分析 -> **向用户询问更多细节**

## 工作流程

1. **需求分析**（本 Skill）- 将业务需求转换为技术实现点
2. **使用cangjie基础skills** - 先使用 `.opencode/skills/` 中的仓颉语言 skills
3. **L1 查询** - 如需进一步查询，使用 `harmonyos-l1-query` skill
4. **L3 查询** - 如需本地文档搜索，使用 `harmonyos-l3-query` skill
5. **构建** - 使用 `harmonyos-build` skill 进行项目构建
6. **迭代进化** - 使用 `harmonyos-evolution` skill 记录和解决问题

## 示例

### 示例 1: UI 组件需求

**用户原始需求**: "创建一个篮球相关的页面"

**技术需求分析**:
- **界面分析**: 需要列表展示球队/球员、图片显示、文本信息、可能的卡片布局
- **数据分析**: 需要数组存储球队数据、可能需要状态管理
- **交互分析**: 列表滚动、可能的点击事件
- **问题类型**: UI/页面开发 🎯

**技术关键词细化**:
- List
- Image
- Text
- Column
- onClick
- State

### 示例 2: 登录页面需求

**用户原始需求**: "创建一个登录页面"

**技术需求分析**:
- **界面分析**: 需要两个输入框(用户名/密码)、登录按钮、表单布局
- **数据分析**: 用户名字符串、密码字符串、可能需要验证状态
- **交互分析**: 输入框.onChange() 监听、按钮.onClick() 提交
- **问题类型**: UI/页面开发 🎯

**技术关键词细化**:
- TextInput
- Button
- Column
