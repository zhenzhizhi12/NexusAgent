# 仓颉普通工程 Evolution 管理技能

## 描述

本技能用于管理仓颉普通工程（非鸿蒙应用）开发过程中的 Evolution.md 文件，记录和查询在开发过程中遇到的重要问题和解决方案，避免重复踩坑。

## 使用场景

本技能在以下场景中使用：

1. **前置学习阶段**：在开始写代码前，先读取 Evolution.md 了解之前踩过的坑，避免重复犯错
2. **经验记录阶段**：在构建过程中遇到的坑需要记录到 Evolution.md 中
3. **错误修复阶段**：每次修复报错前，先读取 Evolution.md 了解之前踩过的坑

## 工作流程

### 1. 前置学习

**时机**：在开始写代码前

**操作**：
1. 读取 Evolution.md 文件
2. 查找与当前任务相关的历史问题
3. 遵循已有的最佳实践，避免重复犯错

**命令**：
```
read C:/Users/zqw/Desktop/common-cangjie/.opencode/skills/cangjie-evolution/Evolution.md
```

### 2. 经验记录

**时机**：在构建过程中遇到新问题并解决后

**操作**：
1. 识别问题的核心症状
2. 记录问题的解决步骤
3. 记录预防措施和最佳实践
4. 将信息追加到 Evolution.md 文件

**记录格式**（与鸿蒙 Evolution.md 格式一致）：

```markdown
## [日期] - [问题描述]

### 问题症状
[描述问题的具体表现和错误信息]

### 问题原因
[分析问题的根本原因]

### 解决方案
[列出解决该问题的具体步骤]

### 预防措施
[列出如何避免此类问题的最佳实践]
```

### 3. 错误修复查询

**时机**：每次修复报错前

**操作**：
1. 读取 Evolution.md 文件
2. 搜索当前遇到的错误信息
3. 如果找到历史记录，按照记录的解决方案处理
4. 如果是新问题，修复后记录到 Evolution.md

## 与其他技能的配合

### 与 common-skill-l1 的配合

- 在使用 common-skill-l1 学习和查询前，先调用本技能读取 Evolution.md
- 在 common-skill-l1 提供的解决方案基础上，结合 Evolution.md 中的经验

### 与 common-skill-l2 的配合

- 在 L1 检索失败或结果不相关时，先检查 Evolution.md
- 在 L3 本地文档搜索后，将发现的新内容补充到 Evolution.md
